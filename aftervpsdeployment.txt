STEP 6 — CREATE CELERY APP
config/celery.py

Create:

import os

from celery import Celery


os.environ.setdefault(
    "DJANGO_SETTINGS_MODULE",
    "config.settings.development",
)

app = Celery(
    "config"
)

app.config_from_object(
    "django.conf:settings",
    namespace="CELERY",
)

app.autodiscover_tasks()
STEP 7 — UPDATE config/init.py

STEP 8 — CREATE FIRST TASK
apps/payments/tasks.py

Create:

from celery import shared_task

from .reconciliation_service import (
    BankReconciliationService,
)


@shared_task
def reconcile_bank_statement_task(
    file_path,
):

    with open(
        file_path,
        "rb",
    ) as file:

        result = (
            BankReconciliationService.process_file(
                uploaded_file=file
            )
        )

    return result
WHY FILE PATH?

Celery workers:

run separately
cannot access request memory.

This is VERY important architecture knowledge.

STEP 9 — UPDATE PAYMENT VIEW
apps/payments/views.py

Replace synchronous reconciliation:

result = (
    BankReconciliationService.process_file(

WITH:

from .tasks import (
    reconcile_bank_statement_task,
)

Then:

import tempfile

Inside POST:

temp_file = tempfile.NamedTemporaryFile(
    delete=False,
)

for chunk in request.FILES["file"].chunks():

    temp_file.write(chunk)

temp_file.close()

reconcile_bank_statement_task.delay(
    temp_file.name
)
THEN REPLACE SUCCESS MESSAGE

WITH:

messages.success(
    request,
    (
        "Bank reconciliation task "
        "started successfully."
    ),
)
STEP 10 — START CELERY WORKER

Open NEW terminal:

IMPORTANT

Now:

Django runs separately
Celery worker runs separately

This is REAL production architecture.
